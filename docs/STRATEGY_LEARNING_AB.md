# STRATEGY LEARNING + A/B 自动决策实施方案

更新时间：2026-03-09

## 目标
将当前“模板随机生成”升级为：
1. 基于历史表现学习可复用内容策略；
2. 同主题多变体 A/B 实验；
3. 用在线决策自动提高优胜策略流量占比。

## 当前问题
1. 候选生成主要依赖固定模板和随机 hook，无法持续学习。
2. 发布默认取第 1 条候选，缺少可解释排序依据。
3. A/B 只停留在“多版本生成”，没有自动胜负判定与流量再分配。

## 总体架构
1. 离线学习层：从历史发布结果提取特征，学习“什么特征更容易获得高互动率”。
2. 在线实验层：同主题生成多个变体，分配曝光并持续更新每个变体的成功概率。
3. 决策执行层：把“下次发什么”从静态规则改为策略引擎输出。

## Phase 1（本周）：先做可解释学习，不上复杂模型
### 1) 数据建模补齐
新增两张表：
- `experiment_runs`：一次实验（主题、发布时间窗、账号、状态）。
- `experiment_arms`：实验变体（标题结构、开头类型、CTA 类型、标签集合、得分、结果）。

说明：`publish_records` 保留原始流水；`experiment_*` 负责实验语义。

### 2) 特征工程
从标题/正文提取低成本特征：
1. 标题长度、是否数字开头、是否问句、关键词命中数。
2. 内容结构（是否有分点、步骤数、CTA 是否明确）。
3. 标签质量（标签数量、标签与主题一致性）。
4. 发布时间上下文（小时、工作日/周末、账号）。

### 3) 训练标签
使用 T+1 或 T+2 聚合指标作为监督信号：
1. `engagement_rate = (likes + comments + collects + shares) / max(exposure, 1)`
2. 二分类标签：是否进入当周前 30%。

### 4) 候选排序改造
在 `03_generate_candidates.py` 增加：
1. 先生成多候选；
2. 再用“规则得分卡”排序（不是随机）；
3. 入库时写入 `score` 与 `feature_snapshot`（JSON 字段可先放在新表）。

## Phase 2（两周内）：上线多臂老虎机自动决策
### 1) Arm 定义
每个 Arm = 固定策略组合，例如：
1. `hook_type`（反常识/清单/复盘/避坑）
2. `structure_type`（3步法/案例法/对比法）
3. `cta_type`（评论领取/收藏复用/私信关键词）

### 2) 在线分配策略
优先 Thompson Sampling（Beta-Bernoulli）：
1. 每个 Arm 维护 `alpha/beta`；
2. 每轮采样后选最大值 Arm；
3. 发布结果回流后更新参数。

成功事件可先定义为：`engagement_rate` 超过该主题最近 7 天中位数。

### 3) 决策约束
1. 探索下限：任何 Arm 占比不低于 10%，防止过早收敛。
2. 安全阈值：连续 N 次低于基线则临时下线该 Arm。
3. 账号隔离：不同账号独立统计，避免相互污染。

## Phase 3（持续）：自动化复盘与策略迭代
1. 周报输出每个 Arm 的胜率、平均互动率、样本量、置信区间。
2. 自动淘汰低效 Arm，自动生成新 Arm（基于高效模板微调）。
3. 对“题材 x 账号 x 时段”做分层策略，减少一刀切。

## 落地改造点（对应现有代码）
1. `scripts/02_analyze.py`
- 增加发布后回收指标与实验维度关联写库。

2. `scripts/03_generate_candidates.py`
- 从“随机模板”改为“多策略生成 + 特征打分 + 排序出队”。

3. `scripts/04_publish_preview.sh` / `scripts/05_publish.sh`
- 从“默认第 1 条”改为“读取策略引擎选中的 candidate_id”。

4. `frontend/scripts/sync-dashboard-data.mjs`
- 增加实验看板字段：Arm 表现、最近 7 天策略变化、自动决策日志。

## 验收标准
1. 两周内达到：
- 同主题至少 3 个 Arm，累计样本 >= 30；
- 自动决策不低于人工基线互动率；
- 周报可解释“为什么本周主推某策略”。

2. 一个月内达到：
- 可稳定自动淘汰低效 Arm；
- 新增策略能在 7 天内完成冷启动评估。
