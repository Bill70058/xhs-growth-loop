# MVP TODO（最小可用版本落地清单）

更新时间：2026-03-09

## 总目标
把当前“可运行原型”推进到“最小可用 MVP”：
1. 有稳定真实回流数据；
2. 决策可追踪、可评估、可回滚；
3. 自动化执行在安全护栏内稳定运行。

## 优先级顺序（必须按序）

### 1. 补数据闭环（最高优先）
- 目标：
  - 提升 `note_id` 覆盖率；
  - 提高真实发布占比（不只停在 preview）；
  - 固定 T+1/T+2 回流采集节奏。
- 完成定义：
  - `note_id_coverage >= 80%`
  - 连续 7 天每天有有效发布回流
  - `pending_reward` 连续下降并低于阈值（例如 < 5）
- 验收指标：
  - `data/runtime/mvp_status.json` 中 `note_id_coverage.ratio`
  - `latest_summary.json` 中 `reward_update.applied/pending`

### 2. 决策版本化与可追溯
- 目标：
  - 决策日志包含 `policy_version/prompt_version/model_version`
  - 支持快速回滚到稳定版本。
- 完成定义：
  - 每轮 cycle 都可追溯“输入->决策->执行->结果”
  - 至少支持 1 次版本切换回滚演练
- 验收指标：
  - `autopilot_milestones` 可关联到版本字段
  - 回滚后 1 轮内可恢复稳定执行

### 3. 在线评估 KPI
- 目标：
  - 每轮自动打分并输出决策收益。
- 完成定义：
  - KPI 至少包含：
    - `pending_reward` 变化
    - `note_id_coverage` 变化
    - `reward_applied` 增长
    - 失败重试成功率
- 验收指标：
  - `LATEST_PROGRESS.md` 与 `mvp_status.json` 持续更新 KPI

### 4. 护栏与背压
- 目标：
  - 防止“实验空转”和风险放大。
- 完成定义：
  - `pending` 高时自动降频生成；
  - 白名单动作、限流、熔断生效；
  - 连续失败自动降级保守策略。
- 验收指标：
  - 高 pending 场景下，生成频率下降且系统不中断

### 5. 检索增强决策（RAG）
- 目标：
  - 决策前检索历史相似成功/失败案例。
- 完成定义：
  - runtime 决策包含“参考历史案例摘要”
  - 决策稳定性优于纯规则/纯提示词
- 验收指标：
  - KPI 至少 1 项显著提升（例如失败重试成功率）

### 6. 策略 A/B 与自动回滚
- 目标：
  - 决策策略本身可 A/B，对比后自动放量或回滚。
- 完成定义：
  - 至少 2 套策略并行 1 周
  - 自动判定胜者并更新流量分配
- 验收指标：
  - 周报显示策略胜负与切换记录
